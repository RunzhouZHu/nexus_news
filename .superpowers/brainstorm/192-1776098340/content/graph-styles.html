Knowledge Graph Visual Styles

Three Sigma.js rendering approaches for Nexus — pick the one that feels right for a news knowledge graph

A — Dark Cosmos

Dark slate background, glowing circular nodes sized by importance, curved animated edges. Dramatic, feels like a starfield of information. Popular in knowledge graph UIs (Obsidian, Roam).

Pros

  • Visually striking
  • Nodes stand out clearly
  • Familiar "knowledge graph" aesthetic

Cons

  • Less text visible at a glance
  • Edge labels harder to read

B — Light Network

Clean white/grey background, solid-color nodes by tag category, straight edges with type labels. Professional, readable — integrates cleanly with the existing sidebar UI.

Pros

  • Matches existing light UI
  • Easy to read labels
  • Clear color-coding by tag

Cons

  • Less "wow factor"
  • Dense graphs can look cluttered

C — Neon Highlight

Dark background with neon accent colors per edge type (red=CAUSED_BY, green=LED_TO, yellow=RELATED_TO). Trending nodes pulse with a glow ring. High-energy, data-journalism feel.

Pros

  • Edge types immediately visible
  • Trending nodes pop
  • Strong visual identity

Cons

  • Can be overwhelming
  • Accessibility concerns
